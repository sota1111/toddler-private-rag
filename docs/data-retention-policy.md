# データ保持およびプライバシー保護方針 (SOT-780)

本プロジェクトでは、保育園関連の機微な情報を扱うため、以下のデータ保護および保持ポリシーを適用しています。

## 1. PII (個人識別情報) の自動マスク (Redaction)

OCRによって抽出されたテキストがデータベースに保存されたり、ログに出力されたりする前に、特定の個人識別情報パターンを自動的に検出してマスクします。

### 対象となるパターン
- **メールアドレス**: `[REDACTED_EMAIL]` に置換
- **電話番号**: 日本の固定電話、携帯電話、フリーダイヤルなどの形式を `[REDACTED_PHONE]` に置換
- **マイナンバー**: 12桁の数字またはハイフン区切りの形式を `[REDACTED_ID]` に置換
- **銀行口座番号**: 「口座」というキーワードの近傍にある7〜8桁の数字、または単独の7〜8桁の数字（日付と誤認されないもの）を `[REDACTED_ACCOUNT]` に置換

### 適用箇所
- `backend/app/routers/attachments.py`: OCR処理後のテキスト保存直前に適用されます。
- システムログ: マスクされた後のテキストのみがログに記録されます。

### 注意事項
この機能はパターンベースの**ベストエフォート**による保護です。自由記述形式の書類に含まれる氏名や住所など、パターン化が困難な全ての個人情報を完全に匿名化することを保証するものではありません。機微な情報の取り扱いには引き続き十分な注意を払ってください。

## 2. データ保持ポリシー

不要なデータ保持を最小限に抑えるため、添付ファイルの保持期間を設定し、期限を超えたデータをパージする機能を提供しています。

### 設定方法
`.env` ファイル（または環境変数）で以下の値を設定します。

```env
# 添付ファイルの保持日数（例: 30日）
# 0 または未設定の場合は無期限（自動削除無効）
ATTACHMENT_RETENTION_DAYS=30
```

### パージ（削除）の実行
期限切れの添付ファイル（物理ファイルおよびデータベースレコード）を削除するには、以下のコマンドを手動またはcron等で定期実行します。

```bash
cd backend
./venv/bin/python -m app.retention
```

### 削除の基準
`Attachment.created_at`（作成日時）を基準とし、現在時刻から `ATTACHMENT_RETENTION_DAYS` を経過したレコードが対象となります。パージが実行されると、紐付いている物理ファイル（ローカルストレージまたはGCS）が削除された後、データベースからレコードが削除されます。

## 3. 既知の限界
- **誤マスク**: 日付や特定の識別番号が電話番号や口座番号と誤認されてマスクされる可能性があります。
- **取りこぼし**: 手書き文字のOCRミスや、特殊な形式の記載により、PIIがマスクされずに残る可能性があります。
- **自由記述**: 文脈に埋め込まれた個人名などは、現在のパターンマッチングでは検出されません。
